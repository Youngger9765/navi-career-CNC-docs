CNC 培訓班數位化方案

一頁看完：為什麼做、做什麼、怎麼做、效益

2026-02-23

現在的問題

  • 手動追學費匯款（6-8hr/梯） — 一筆一筆對帳
  • 手動建 Classroom + Drive + Meet（8-10hr/梯） — 每梯重複建 10+ 個資源
  • 補課影片上下架 x13 堂（13-20hr/梯） — 上傳 YouTube → 貼連結 → 3 週後手動下架
  • BCC 群發通知信（8-12hr/梯） — 報名確認、課前通知、認證提醒...
  • 附件 A-E 逐一檢查（4-6hr/梯） — 打開每個學員的 5 份文件核對

4 梯/年 = 240-360 小時/年 ≈ 1.5 個全職人力只在做重複操作

學員資料散落在 Google Forms、Sheets、Classroom、LINE、開課快手 5 個地方

做什麼：三階段上線

Phase 1（8 週）— 報名+付款自動化

學員線上報名 → 線上付款 → 自動收到確認信
行政後台一眼看到誰報了、誰付了

→ 行政省 40-50% 時間
Phase 2（8 週）— 上課到拿證全線上

課程管理、督導線上預約、認證自動審核、證書自動產生

→ 行政省 60-70% 時間
Phase 3（6 週）— 數據+內容管理

補課影片自動上下架、開課快手替代、營運儀表板
學員 CRM、校友管理

→ 累計省 70-83% 時間

Phase 1：8 週詳細展開

W1：環境建置 + 基礎架構（GCP 部署、CI/CD、專案骨架）
W2：帳號系統 + OAuth（Google 登入、Email 註冊、身分驗證、權限）
W3：報名表單（學員報名表 → 存 DB、管理後台報名列表）
W4：金流串接（信用卡 + ATM、Webhook 回調、付款狀態更新）
W5：金流進階 + 發票（分期、退款、電子發票、每日對帳）
W6：Email 通知系統（SendGrid、4 種通知信範本）
W7：管理後台完善（梯次建立、報名/付款一覽、匯出 Excel）
W8：測試 + 上線（端到端測試、UAT、正式上線、操作教學）

不做的事（明確排除）

  • LINE 整合 — 群組繼續用，正式通知走 Email
  • 包裹寄送自動化 — 每年只 4 次，手動就好
  • 進階 Email 行銷自動化 — SendGrid 直接操作足夠
  • API 對接（ChinaNCDA） — 現階段無迫切需求
  • 進階報表（同期群分析、LTV） — 基礎儀表板足夠
  • Mobile App — 響應式網頁就夠
