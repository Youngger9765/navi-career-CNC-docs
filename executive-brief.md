# CNC 培訓班數位化方案｜決策摘要

> 一頁看完：為什麼做、做什麼、怎麼做、效益

---

## 現在的問題

行政助教每辦一梯培訓班，要花 **60-90 小時**做手動作業：

| 問題 | 每梯耗時 | 現況 |
|------|---------|------|
| 手動追學費匯款 | 6-8hr | 一筆一筆對帳 |
| 手動建 Classroom + Drive + Meet | 8-10hr | 每梯重複建 10+ 個資源 |
| 補課影片上下架 x13 堂 | 13-20hr | 上傳 YouTube → 貼連結 → 3 週後手動下架 |
| BCC 群發通知信 | 8-12hr | 報名確認、課前通知、認證提醒... |
| 附件 A-E 逐一檢查 | 4-6hr | 打開每個學員的 5 份文件核對 |

4 梯/年 = **240-360 小時/年**的行政時間，相當於 **1.5 個全職人力**只在做重複操作。

學員資料散落在 Google Forms、Sheets、Classroom、LINE、開課快手 **5 個地方**，沒有統一來源。

---

## 做什麼

建一個培訓管理平台，分兩階段上線：

### Phase 1（8 週）— 報名+付款自動化

學員線上報名 → 線上付款 → 自動收到確認信。行政後台一眼看到誰報了、誰付了。

**行政省 40-50% 時間**

<details>
<summary>Phase 1 週次展開</summary>

| 週次 | 工作內容 | 交付物 |
|------|---------|--------|
| W1 | 環境建置 + 基礎架構 | GCP 部署、CI/CD、專案骨架 |
| W2 | 帳號系統 + OAuth | Google OAuth 登入、Email/Password 註冊、JWT、權限 |
| W3 | 報名表單（前端+後端） | 學員報名表 → 存 DB、管理後台報名列表 |
| W4 | 金流串接 | 信用卡 + ATM、Webhook 回調、付款狀態更新 |
| W5 | 金流進階 + 發票 | 分期付款、退款、電子發票、每日對帳 |
| W6 | Email 通知系統 | SendGrid 串接、4 種通知信範本 |
| W7 | 管理後台完善 | 梯次建立、報名/付款一覽、匯出 Excel、對帳報表 |
| W8 | 測試 + 上線 | 端到端測試、UAT、正式上線、操作教學 |

</details>

### Phase 2（14 週）— 上課到拿證全線上+基礎數據

一鍵建新梯次、學員統一入口、督導線上預約、認證自動審核、證書自動產生、營運儀表板即時掌握關鍵數據。

**累計省 70-83% 時間**

---

## 時程規劃

| | Phase 1 | Phase 2 |
|--|---------|---------|
| 時間 | **8 週** | **14 週** |
| 範圍 | 報名+付款+通知 | 課程+督導+認證+CRM |

Phase 2 在 Phase 1 成功跑完一梯後才啟動，有明確停損點。

Phase 2 包含基礎 CRM（儀表板、學員列表、校友管理）。進階功能（Email 行銷、API 對接、進階報表）暫不做，等規模到 300+ 人/年再評估。

---

## 賺多少（ROI）

| 效益 | 估算 |
|------|------|
| **省行政人力** | 每年省 200-280 小時 ≈ NT$80-112K/年 |
| **數據決策** | 即時掌握營運指標，節省 Excel 手動彙整 ≈ NT$50K/年 |
| **學員追蹤** | CRM 提升學員續約率與校友管理效率 |
| **支撐擴張** | 系統到位後可從 4 梯擴到 6-8 梯（不加人） |
| **增量營收** | 多 2-4 梯 ≈ NT$6-12M 增量營收/年 |
| **回收期** | 保守 18 個月，樂觀 6-9 個月 |

## 不做的事

- 不做 LINE 整合（群組繼續用，正式通知走 Email）
- 不做包裹寄送自動化（每年只 4 次，手動就好）
- 不做進階 Email 行銷自動化（SendGrid 直接操作足夠）
- 不做 API 對接（ChinaNCDA，現階段無迫切需求）
- 不做進階報表（同期群分析、LTV，基礎儀表板足夠）
- 不做 Mobile App（響應式網頁就夠）
